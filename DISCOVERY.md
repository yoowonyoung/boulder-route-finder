## 🔍 핵심 문제

클라이밍장에서 루트를 보고 최적의 등반 방법(베타)을 알고 싶음

---

## 👤 사용자 & 목적

- **대상**: 볼더링 클라이머 (초중급)
- **목적**: 개인 사용 / 추후 공유 가능

---

## 🧪 검증 결과

### 시도 1: Vision AI로 홀드 자동 감지
```
❌ 실패
- 손상된/먼지 묻은 홀드를 검정색으로 오인식
- 조명, 각도에 따라 색상 구분 어려움
- 정확도가 실사용하기엔 부족
```

### 시도 2: 반자동 방식 (채택 ✅)
```
✅ 채택
사진 업로드 → 사용자가 홀드 직접 마킹 → AI가 베타 추천
- 홀드 감지 문제 우회
- 사용자가 정확한 루트 지정
- AI는 베타 추천에 집중
```

---

## 🎯 v1 범위

### 필수 (v1)
| 기능 | 설명 |
|------|------|
| 이미지 업로드 | 클라이밍 벽 사진 |
| 홀드 마킹 UI | 터치/클릭으로 홀드 위치 순서대로 마킹 |
| 시작/탑 표시 | 시작점, 종료점 구분 |
| 베타 추천 | 홀드 위치 기반 등반 시퀀스 제안 |

### 나중에 (v2+)
| 기능 | 설명 |
|------|------|
| 홀드 타입 선택 | 슬로퍼/크림프/저그/포켓 등 |
| 신체 조건 입력 | 키, 암 스팬 반영 |
| 베타 히스토리 | 저장 및 공유 |
| 난이도 추정 | 홀드 배치 기반 그레이드 예측 |

---

## 🔧 기술 스택 제안

- **Frontend**: React + TypeScript + Tailwind
  - Canvas 또는 SVG로 마킹 UI 구현
- **Backend**: Python + FastAPI (또는 LLM API 직접 호출)
  - 볼더링 도메인 지식 프롬프트
  - 홀드 위치 분석 → 베타 생성

---

## 📚 도메인 지식 (Backend에 포함)

```markdown
### 홀드 타입
- 저그(Jug): 큰 손잡이, 쉬움
- 크림프(Crimp): 작은 홀드, 손가락 끝
- 슬로퍼(Sloper): 경사진 홀드, 마찰력
- 핀치(Pinch): 엄지로 조여서
- 포켓(Pocket): 손가락 1-3개

### 기본 무브
- 정적 무브: 천천히 이동
- 다이노(Dyno): 점프
- 힐훅/토훅: 발 걸기
- 플래깅: 균형용 발 교차
- 드롭니: 무릎 꺾어 무게중심 이동

### 루트 리딩 원칙
- 아래서 위로 시퀀싱
- 쉬는 구간 파악
- 크럭스(핵심 구간) 미리 파악
- 발 위치가 손만큼 중요
```
